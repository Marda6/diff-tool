# Diff & Merge: цвета и метки обратной связи

Справочник по цветовой схеме инструментов сравнения кода (diff/merge) и
визуальным маркерам, показывающим состояние изменений. Единого стандарта нет —
конкретные оттенки зависят от инструмента и темы, но логика общая.

---

## 1. Цвета различий между файлами

| Цвет | Что означает | Пояснение |
|------|--------------|-----------|
| 🟢 **Зелёный** | Строка добавлена | Есть в новом/правом файле, отсутствует в старом/левом |
| 🔴 **Красный** | Строка удалена | Была в старом/левом файле, отсутствует в новом/правом |
| 🟡 **Жёлтый** (или 🔵 синий) | Строка изменена | Есть в обоих файлах, но содержимое отличается |
| ⚪ **Серый / без подсветки** | Строки одинаковые | Совпадают в обоих файлах |

### Уникальная строка (есть только в одном файле)

Цвет зависит от того, **в какой панели** находится строка:

- Только **слева** (справа нет) → чаще **красный** («удалённая»)
- Только **справа** (слева нет) → чаще **зелёный** («добавленная»)

> Цвет определяется не статусом read-only/редактируемый, а тем, в какой панели
> строка и трактует ли инструмент её как «исходную» или «целевую».

---

## 2. Режим «один файл редактируемый, второй read-only»

Логика подсветки та же, меняется только точка отсчёта:

- **Read-only файл** = база/эталон, с которым сверяемся.
- **Редактируемый файл** = тот, что приводим в нужный вид.

Особенности:

- Подсветка обновляется **в реальном времени** — по мере правок блоки
  отличий исчезают, когда строки совпадут с эталоном.
- Read-only панель обычно помечена: серый фон, иконка замка 🔒.
- Стрелки/кнопки переноса изменений работают **только в одну сторону** —
  в редактируемый файл.

---

## 3. Что происходит после merge (слева → направо)

Когда изменение перенесено и строки совпали, **яркий цвет исчезает** — блок
становится нейтральным (серым/бледным). Само исчезновение цвета = сигнал
«здесь уже синхронизировано».

Чтобы не потерять, что блок был тронут, инструменты дают доп. маркеры:

| Маркер | Где | Что показывает |
|--------|-----|----------------|
| **Звёздочка / точка** | Заголовок вкладки (`file.txt *`) | Файл изменён, но **не сохранён** |
| **Цветная полоса в gutter** | Слева от номеров строк | Разница с **последним сохранённым** состоянием (не между файлами) |
| **Засечки на overview / minimap** | Вертикальная полоса-обзор сбоку | Места, где была активность |
| **Изменённое имя файла** | Заголовок | Цвет/пометка «modified» |

---

## 4. Важный нюанс

После merge **и сохранения** файлы по этому блоку совпадают → подсветки
«отличий» там не будет в принципе (отличий нет).

Поэтому, чтобы «не забыть, что тут правил», полагаться нужно **не на
diff-цвета** (они исчезнут), а на:

- историю / undo-стек инструмента;
- маркеры несохранённых изменений (до сохранения);
- систему контроля версий (Git и т.п.) — покажет изменение относительно коммита;
- устойчивая пометка: комментарий в коде или **закладка (bookmark)**, если
  инструмент поддерживает.

---

## 5. Краткая шпаргалка

- 🟢 зелёный = добавлено · 🔴 красный = удалено · 🟡 жёлтый = изменено · ⚪ серый = одинаково
- Уникальная строка: слева → красный, справа → зелёный
- Read-only = эталон (замок 🔒), перенос только в редактируемый файл
- После merge цвет отличия **пропадает**; «изменено, но не сохранено» = `*` в заголовке + полоса в gutter
- После сохранения отслеживай изменения через Git, а не через diff-цвета
